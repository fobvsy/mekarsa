## 7. Fitur Utama

### 7.1 Halaman Beranda

Halaman beranda menjadi halaman utama yang menampilkan identitas Mekarsa sebagai coffee bar. Konten utama yang ditampilkan meliputi hero section, tagline, foto/banner utama, menu unggulan, kategori produk, artikel terbaru, galeri singkat, testimoni, lokasi, dan tombol WhatsApp.

Data pada halaman beranda harus diambil dari database, bukan ditulis secara hardcode di HTML.

### 7.2 Profil Usaha

Halaman profil berisi informasi tentang Mekarsa, konsep usaha, alamat, karakter brand, dan penjelasan bahwa Mekarsa berfokus pada coffee bar dengan layanan shoe clean sebagai layanan pendukung.

### 7.3 Katalog Produk Coffee Bar

Fitur katalog produk menjadi fitur utama website. Setiap produk coffee memiliki informasi berupa nama produk, kategori, deskripsi, harga, gambar, status ketersediaan, dan label khusus jika produk termasuk menu unggulan atau signature menu.

Contoh kategori produk:

1. Coffee.
2. Non-Coffee.
3. Signature Drinks.
4. Manual Brew.
5. Snack.
6. Promo Menu.

Berdasarkan daftar menu Mekarsa yang tersedia, produk coffee awal yang akan dimasukkan ke database dan ditampilkan pada katalog adalah sebagai berikut:

### 7.4 Detail Produk Coffee

Halaman detail produk menampilkan informasi lebih lengkap mengenai satu produk, seperti nama, foto, harga, deskripsi, kategori, rekomendasi penyajian, status ketersediaan, dan tombol pesan melalui WhatsApp.

### 7.5 Produk Unggulan dan Promo

Website dapat menampilkan produk unggulan atau promo pada halaman beranda dan halaman katalog. Produk unggulan digunakan untuk menonjolkan menu yang direkomendasikan oleh Mekarsa, sedangkan promo digunakan untuk menampilkan informasi potongan harga atau paket tertentu.

### 7.6 Form Pemesanan Coffee

Fitur form pemesanan digunakan agar pelanggan dapat mengirim data pesanan secara sederhana. Pelanggan dapat mengisi nama, nomor WhatsApp, produk yang dipilih, jumlah produk, catatan tambahan, dan metode pengambilan. Setelah form dikirim, data pesanan disimpan ke database dan pelanggan dapat diarahkan ke WhatsApp admin dengan format pesan otomatis.

### 7.7 Blog atau Artikel

Fitur blog/artikel digunakan sebagai media edukasi dan promosi. Artikel dapat membahas topik seputar coffee, lifestyle, rekomendasi menu, promo, event kecil, atau informasi terbaru Mekarsa.

Contoh isi artikel:

1. Rekomendasi menu coffee untuk menemani tugas kuliah.
2. Perbedaan kopi susu dan manual brew.
3. Cara memilih menu coffee sesuai selera.
4. Menu non-coffee untuk pelanggan yang tidak minum kopi.
5. Promo terbaru Mekarsa Coffee Bar.
6. Cerita di balik signature menu Mekarsa.

### 7.8 Galeri dan Testimoni

Galeri digunakan untuk menampilkan foto produk coffee, suasana tempat, pelanggan, event kecil, dan dokumentasi kegiatan. Testimoni digunakan untuk menampilkan ulasan pelanggan agar pengunjung lebih percaya terhadap kualitas produk dan tempat Mekarsa.

### 7.9 Layanan Shoe Clean sebagai Pendukung

Layanan shoe clean tetap ditampilkan secara ringkas agar pengunjung mengetahui identitas lengkap Mekarsa. Halaman ini dapat berisi daftar layanan perawatan sepatu, deskripsi singkat, estimasi harga, dan tombol WhatsApp. Namun, prioritas visual dan konten website tetap diarahkan pada produk coffee.

### 7.10 Kontak dan Lokasi

Halaman kontak menampilkan alamat lengkap **Jl. Pabelan I, Gatak, Pabelan, Kec. Kartasura, Kabupaten Sukoharjo, Jawa Tengah 57169**, nomor WhatsApp **085933504096**, Instagram **@mekarsaa**, jam operasional, dan peta lokasi. Nomor WhatsApp digunakan dalam format tautan **`https://wa.me/6285933504096`** agar pengunjung dapat langsung menghubungi admin.

### 7.11 Login Admin

Admin memiliki halaman login khusus. Password admin disimpan dalam bentuk hash menggunakan `password_hash()` pada PHP. Login menggunakan session PHP. Seluruh halaman admin dilindungi sehingga pengguna yang belum login akan diarahkan kembali ke halaman login.

### 7.12 Panel Admin

Panel admin digunakan untuk mengelola seluruh konten website. Admin dapat melakukan tambah, lihat, edit, dan hapus data pada produk coffee, kategori produk, artikel, kategori artikel, galeri, testimoni, pesanan, layanan pendukung, dan pengaturan website.

Dashboard admin menampilkan ringkasan data seperti jumlah produk coffee, jumlah kategori, jumlah artikel, jumlah testimoni, jumlah pesanan, dan jumlah galeri.

## 13. Rancangan Database

Database menggunakan MySQL. Seluruh data utama yang ditampilkan pada website diambil dari database, bukan hardcode di HTML.

### 13.1 Daftar Tabel

| No | Nama Tabel | Fungsi |
|----|------------|--------|
| 1 | `admins` | Menyimpan akun admin. |
| 2 | `product_categories` | Menyimpan kategori produk coffee. |
| 3 | `products` | Menyimpan data produk coffee bar. |
| 4 | `article_categories` | Menyimpan kategori artikel. |
| 5 | `articles` | Menyimpan data blog/artikel. |
| 6 | `gallery` | Menyimpan data foto produk, suasana tempat, dan dokumentasi. |
| 7 | `testimonials` | Menyimpan testimoni pelanggan. |
| 8 | `orders` | Menyimpan data pesanan pelanggan. |
| 9 | `order_items` | Menyimpan detail produk yang dipesan. |
| 10 | `support_services` | Menyimpan data layanan shoe clean sebagai layanan pendukung. |
| 11 | `settings` | Menyimpan informasi umum website. |

### 13.2 Relasi Tabel

1. Tabel `products` berelasi dengan `product_categories` melalui `category_id`.
2. Tabel `articles` berelasi dengan `article_categories` melalui `category_id`.
3. Tabel `articles` berelasi dengan `admins` melalui `author_id`.
4. Tabel `order_items` berelasi dengan `orders` melalui `order_id`.
